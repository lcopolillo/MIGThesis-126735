## Setup

**Python requirement:** 3.12.

```bash
# 1. Create virtual environment with Python 3.12
python3.12 -m venv .venv
source .venv/bin/activate        # macOS / Linux
# .venv\Scripts\activate         # Windows

# 2. Install dependencies
pip install -r requirements.txt

# 3. Install the Portuguese spaCy model (required for preprocessing)
python -m spacy download pt_core_news_sm

# 4. Create the data directories (gitignored — not included in the repository)
mkdir -p data/input data/output data/logs

# 5. Place your source PDF files in data/input/

# 6. Create a .env file at the project root (gitignored — must be created manually)
```

The `.env` file controls three path variables (relative paths are resolved from the project root):

```
INPUT_ROOT="data/input"
OUTPUT_ROOT="data/output"
LOG_ROOT="data/logs"
```

> **Note:** `data/input/`, `data/output/`, `data/logs/`, and `.env` are all gitignored and must be set up locally. The repository contains only source code and configuration files.

### `src/utils.py` — public API

| Function | Description |
|---|---|
| `load_corpus(output_root)` | Loads all latest `_cleaned_vN.csv` files into a single line-level DataFrame |
| `tokenize(text)` | Lemmatises one string; returns nouns/adjectives ≥ 3 chars, stopwords removed |
| `tokenize_corpus(corpus)` | Applies `tokenize()` to every line; adds `tokens` and `token_count` columns |
| `build_slide_page_docs(corpus)` | Aggregates tokens to page-level documents (one document per slide page) |
| `build_slide_deck_docs(corpus)` | Aggregates tokens to deck-level documents (one document per PDF file) |

## Running the Pipeline

### 1. Extract and clean slides

```bash
python src/extractor.py
```

Reads PDFs from `INPUT_ROOT`, writes versioned `_vN.csv` raw files and `_vN_cleaned_vN.csv` cleaned files to `OUTPUT_ROOT`. Skips re-extraction if a raw CSV already exists for a given PDF.

**Cleaning steps applied in order:**
1. Cast `text` to string, fill nulls
2. Remove empty / whitespace-only lines
3. Remove duplicate lines (keep first occurrence)
4. Remove lines containing only punctuation
5. Remove lines of isolated single characters
6. Remove lines of isolated numbers
7. Strip URLs (`http://`, `https://`, `www.`) from lines
8. Strip email addresses from lines
9. Remove lines that became empty after URL / email stripping
10. Remove lines with fewer than 3 alphabetic characters
11. Remove table-row artifacts where digits dominate (alpha ratio < 40 %)

### 2. Exploratory Analysis

Open `src/notebooks/exploratory_analysis.ipynb` and run all cells.  
Uses the `.venv` kernel (Python 3.12).

Sections: Setup & Data Loading → Document-Level Analysis → Term Frequency → TF-IDF → Word Cloud.

### 3. LDA Baseline

Open `src/notebooks/lda.ipynb` and run all cells.

Sections: Setup & Data Loading → Corpus Preparation → Model Training → Topic Inspection → Evaluation (C_v coherence + log-perplexity).
